# 토스증권 Server Developer (Platform) - 자기소개서

## 지원 동기

"토스증권 내부 개발자의 개발환경과 경험을 더 좋게 만드는 분야에 기여하고 싶습니다."

토스증권 서버 플랫폼팀이 "사일로 조직에서 필요로 하는 공통 기능, 프레임워크를 만드는 팀"이라는
소개를 읽고, 제 경험과 가치관이 완벽하게 일치한다고 느꼈습니다.

저는 개인 프로젝트에서 **"다른 개발자가 실수 없이 쓸 수 있는 공통 프레임워크"**를 설계하는
역량을 길러왔습니다. LogicExecutor, TieredCache, LockStrategy 등을 개발하면서
"단순히 코드를 재사용하는 것이 아니라, 팀의 코딩 스타일과 아키텍처를 정의하는 것"이
플랫폼 엔지니어의 핵심 역량임을 배웠습니다.

## 핵심 경험 1: 공통 프레임워크 설계 역량

### LogicExecutor 템플릿 개발

**문제**: 비즈니스 로직에 try-catch가 난무하고, 예외 처리가 일관되지 않음

**해결**: 6가지 실행 패턴과 TaskContext 강제로 "다른 개발자가 실수 없이 쓸 수 있는 API" 설계

- `execute()`, `executeOrDefault()`, `executeWithRecovery()` 등 상황별 명확한 구분
- TaskContext로 도메인, 작업명, 식별자 강제 → 구조화된 로그 자동 생성
- ExecutionPipeline AOP 연동으로 MDC, 메트릭 자동 처리

**성과**: try-catch 30% 제거, 디버깅 시간 50% 단축

**배운 점**:
> "공통 라이브러리는 팀의 코딩 스타일을 정의한다.
> 인터페이스 설계는 '이 메서드를 언제 써야 하는지'가 명확해야
> 다른 개발자가 실수 없이 사용할 수 있다."

이 경험이 토스증권 플랫폼팀의 "공통 기능을 스스로 도출해서 만드는" 업무와
직접 연결된다고 확신합니다.

---

## 핵심 경험 2: 데이터 무결성과 금융 시스템

### Outbox 패턴 Triple Safety Net

**문제**: Transactional Outbox에서 Stalled Event가 무한 루프

**해결**: Content Hash(SHA-256), Stalled Recovery, DLQ 이동의 3중 방어

- **무결성**: SHA-256으로 데이터 변조 감지
- **멱등성**: requestId UNIQUE 제약으로 중복 처리 방지
- **감사 가능성**: DLQ에 실패 원인과 payload 보존

**성과**: 216만 이벤트 99.997% 성공

**금융 시스템과의 연결**:
> "금융 시스템에서는 무결성, 멱등성, 감사 가능성이 필수다.
> Content Hash는 주문/정산 데이터 위변조를 방지하고,
> requestId는 결제 중복 승인을 방지한다.
> DLQ는 규제 기관 제출용 증거가 된다."

토스증권이 "증권 서비스들이 안정적으로 운영될 수 있도록 하는" 목표와
제 경험이 일치한다고 생각합니다.

## 핵심 경험 3: 장애 탄력성과 모니터링

### Cache Stampede 방어

**문제**: 부하 테스트에서 캐시 만료 시 DB 호출 폭증 (1000 reqs → 1000 API calls)

**해결**: Single-flight 패턴 + TieredCache (L1 Caffeine + L2 Redis)

**성과**: DB 호출 99% 감소, p99 레이턴시 92% 개선

**배운 점**:
> "부하 테스트는 코드 리뷰로는 발견할 수 없는 숨은 병목을 드러낸다.
> '이론적인 안전성'보다 '실제 장애 주입 테스트'가 신뢰할 수 있다."

### 모니터링 시스템 구축

**문제**: 알림 피로로 중요한 알림을 놓침, 메트릭만 보고 장애 원인 파악 어려움

**해결**:
- 심각도별 채널 분리 (CRITICAL/WARNING/INFO)
- 즉시 실행 가능한 액션 연결 (Thread Pool 90% → AWS 스케일링 버튼)
- TaskContext 기반 구조화된 로그

**성과**: 알림 피로 70% 감소, 장애 조사 시간 80% 단축

목표에 기여할 수 있습니다.

### 3. 장애 탄력성
20개 카오스 시나리오 기반 장애 탄력성 검증, Cache Stampede 92% 개선,
CI Pass Rate 99.7% 달성 등 "안정성을 위한 구조"를 설계하는 경험이 있습니다.

### 4. 모니터링과 운영

TaskContext 기반 구조화된 로그 등 "운영하기 쉬운 구조"를 만드는 경험이 있습니다.
